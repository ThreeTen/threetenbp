Access to date and time using fields and units, plus additional value type classes.

This package expands on the base package to provide additional functionality for more powerful use cases. Support is included for:

  • Units of date-time, such as years, months, days and hours
  • Fields of date-time, such as month-of-year, day-of-week or hour-of-day
  • Additional date time value types, such as Year, YearMonth and MonthDay
  • Date-time adjustment functions
  • Different definitions of weeks

Fields and Units

Dates and times are expressed in terms of fields and units. A unit is used to measure an amount of time, such as years, days or minutes. All units implement {@link javax.time.calendrical.PeriodUnit}. The set of well known units is defined in {@link javax.time.calendrical.LocalPeriodUnit}, for example, {@link javax.time.calendrical.LocalPeriodUnit#DAYS}. The unit interface is designed to allow applications to add their own units.

A field is used to express part of a larger date-time, such as year, month-of-year or second-of-minute. All fields implement {@link javax.time.calendrical.DateTimeField}. The set of well known fields are defined in {@link javax.time.calendrical.LocalDateTimeField}, for example, {@link javax.time.calendrical.LocalDateTimeField#HOUR_OF_DAY}. Additional fields are defined by {@link javax.time.calendrical.JulianDayField} and {@link javax.time.calendrical.QuarterYearField}. The field interface is designed to allow applications to add their own fields.

This package provides tools that allow the units and fields of date and time to be accessed in a general way most suited for frameworks. {@link javax.time.calendrical.DateTime} provides the abstraction for date time types that support fields. Its methods support getting the value of a field, creating a new date time with the value of a field modified, and extracting another date time type, typically used to extract the offset or time-zone.

One use of fields in application code is to retrieve fields for which there is no convenience method. For example, getting the day-of-month is common enough that there is a method on {@code LocalDate} called {@code getDayOfMonth()}. However for more unusual fields it is necessary to use the field. For example, {@code date.get(LocalDateField.ALIGNED_WEEK_OF_MONTH)}. The fields also provide access to the range of valid values.

Value types

This package also contains some additional value types. This are used to ensure that only the right date-time information is stored. The classes are:

  • {@link javax.time.calendrical.Year}
  • {@link javax.time.calendrical.YearMonth}
  • {@link javax.time.calendrical.MonthDay}
  • {@link javax.time.calendrical.QuarterOfYear}

Adjustment

A key part of the date-time problem space is adjusting a date to a new, related value, such as the "last day of the month", or "next Wednesday". These are modeled as functions that adjust a base date-time. The functions implement {@link javax.time.calendrical.DateTimeAdjuster} and operate on {@link javax.time.calendrical.AdjustableDateTime}. A set of common functions are provided in {@link javax.time.calendrical.DateTimeAdjusters}, for example, to find the next date time having a requested {@link javax.time.calendrical.DateTimeAdjusters#nextOrCurrent(DayOfWeek)}.

Weeks

Different locales have different definitions of the week. For example, in Europe the week typically starts on a Monday, while in the US it starts on a Sunday. The {@link javax.time.calendrical.WeekContext} class models this distinction.
